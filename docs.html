Build Docs — Multi-Agent System (OpenAI Agents SDK)

Hands-free demo · OpenRouter backend · 3 patterns · FastAPI

Architecture

LayerFileRole

Pattern 1 — Handoffs

TriageAgent has 3 handoff() tools. The SDK transfers control to Billing / Technical / Sales specialists, preserving history.
handoffs=[
  handoff(billing_agent,  tool_name_override="route_to_billing"),
  handoff(technical_agent,tool_name_override="route_to_technical"),
  handoff(sales_agent,    tool_name_override="route_to_sales"),
]

Pattern 2 — Manager + Workers

Workers wrapped as tools via agent_as_tool(). The ManagerAgent calls research → write → review in order, then synthesizes.
manager_agent = Agent(name="ManagerAgent", tools=[research_tool, writer_tool, reviewer_tool])

Pattern 3 — Agents-as-Tools (pipeline)

CoordinatorAgent calls a translator and a sentiment agent as tools on the same input, presenting both results.
coordinator_agent = Agent(name="CoordinatorAgent",
    tools=[translate_tool, sentiment_tool])

Run it (real steps)

1. Create venv & install: 
uv venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
2. Add your key: copy .env.example → .env and set OPENROUTER_API_KEY from openrouter.ai/keys.
3. Launch (the run script unsets PYTHONPATH to dodge a Windows venv clobber bug): 
run.bat        # Windows
# or
./run.sh        # macOS/Linux
4. Open http://127.0.0.1:8000 — it auto-runs all 3 patterns on load. Inspect the browser console to watch each agent stream in.

API

GET /api/run?pattern=all&prompt=...     # run every pattern
GET /api/run?pattern=handoffs&prompt=... # single pattern
GET /api/health                          # {status, has_key}

Notes

  • Per-agent fallback: runner.py tries DEFAULT_MODEL then each FALLBACK_MODELS on rate-limit/429.
  • Tracing is disabled (set_tracing_disabled) to keep the demo output clean — re-enable to export traces to OpenAI/OpenRouter.
  • Works with any OpenAI-compatible backend by swapping OPENROUTER_BASE_URL.
